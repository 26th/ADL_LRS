OAuth Clients

  1. By default, only HTTP Auth is enabled. To enable OAuth, set the 'OAUTH_ENABLED' flag to true in settings.py
  2. Visit our client registration page
  3. Submit the name and description of the client you wish to authorize
  4. Once submitted, you will receive a client ID and client secret. Use these with the appropriate OAuth headers to perform the initial handshake to gain your temporary token. ({% url oauth_provider.views.request_token %}) NOTE: If you are not familiar with OAuth, please consult the spec and the OAuth 1.0 RFC
  5. You'll then receive your request token. Use this when visiting the authentication page ({% url oauth_provider.views.user_authorization %}). An LRS OAuth ID will be issued to you for you to POST back to the authentication page from your client
  6. Once the client is authenticated, send the appropriate OAuth headers and values from your token to gain an access token.({% url oauth_provider.views.access_token %})
  7. You can now send this access token in with the OAuth headers when making calls to the LRS system
